# 控制器

在机器人技术和自动化领域，控制回路（Control Loop）用于调节系统状态。例如：房间里的温度自动调节器。

当设置了温度，即设定了温度调节器的期望状态（*Desried State*），而房间的实际温度则为当前状态（*Current State*）。通过对设备的开关控制，温度自动调节器让其当前状态接近期望状态。

在 Kubernetes 中，控制器通过监控集群的公共状态，并致力于将当前状态转变了期望的状态。

## 控制器模式

一个控制器至少跟踪一种类型的 Kubernetes 资源。这些 对象 有一个代表期望状态的的 `spec` 字段。该资源的控制器负责确保其当前状态接近期望状态。

控制器可能会自动执行操作，更常见的是，在 Kubernetes 中，控制器会向 apiserver 发送带有异常内容的消息。

### 通过 apiserver 来控制

Job 控制是一个 Kubernetes 内置控制器的例子。内置控制通过和集群 apiserver 交互来管理状态。

Job 是一种 Kubernetes 资源，它运行一个或多个 Pod，任务执行完成后停止。（一旦被调度了，对 `kubelet` 来说 Pod 对象就会变成期望状态的一部分）

在集群中，当 Job 控制器拿到新任务时，它会保证一组 Node 节点上的 `kubelet` 可以运行正确数量的 Pod 来完成工作。Job 控制器不会自己运行任何 Pod 或者容器。Job 控制器是通知 apiserver 来创建或者移除 Pod。
